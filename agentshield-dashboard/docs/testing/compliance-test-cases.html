00 Prerequisites & Setup

🔧 Environment Requirements
  • AgentShield backend running (local npm start or deployed at GCP)
  • PostgreSQL with compliance migrations applied (007_compliance.sql, 012_compliance_rules.sql)
  • At least one agent registered (e.g., GPT-4 Analyst or MCP Everything Server)
  • Logged in as admin or editor role
  • Dashboard accessible at http://localhost:5174 or production URL
  • Compliance module enabled in Settings → Module Toggles
⚠️ Before You Begin Ensure the Compliance module is enabled in Settings → Module Toggles. The module has 4 tabs: 📋 Configs, 🔍 Scans, 📜 History, ⚙️ Compliance Rules. Seed data includes built-in rules for SOX, HIPAA, GDPR, and PCI-DSS.

API Base URL

# Local development
BASE_URL=http://localhost:3000/api/v1/admin

# Production (GCP)
BASE_URL=https://<your-cloud-run-url>/api/v1/admin

Authentication Header

# Get token via login
curl -X POST $BASE_URL/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@agentshield.local","password":"admin123"}'

# Use token in subsequent requests
AUTH="Authorization: Bearer <your-jwt-token>"

01 Config Management (CRUD)

A Compliance Config defines what framework (SOX/HIPAA/GDPR/PCI-DSS) to check, which agent and workflow to target, the sample rate, and retention period. Configs are managed in the 📋 Configs tab.

TC-C01 Create a SOX Compliance Config (All Agents) High ▼

Objective: Create a new SOX compliance configuration targeting all agents.

📌 Via Dashboard UI

  1. Navigate to Compliance in the sidebar.
  2. Ensure the 📋 Configs tab is active.
  3. Click the "+ Add Config" button.
  4. Fill in the form fields as shown in the input table below.
  5. Leave Target Agent as "— All Agents —".
  6. Leave Target Workflow as "— All Workflows —".
  7. Click "Create".

📌 Via API

curl -X POST $BASE_URL/compliance/configs \
  -H "Content-Type: application/json" \
  -H "$AUTH" \
  -d '{
    "name": "SOX Financial Audit - All Agents",
    "framework": "sox",
    "sampleRate": 0.10,
    "retentionDays": 2190,
    "appliesTo": { "agents": [], "workflows": [] }
  }'

03 Compliance Scans

The 🔍 Scans tab is the central place to run compliance checks. Select a config, optionally upload custom samples, and run the scan to see live results with per-rule pass/fail details.

TC-S01 Run Scan — Select Config and Execute High ▼

Objective: Run a compliance scan by selecting a config and clicking Run Scan.

📌 Via Dashboard UI

  1. Navigate to Compliance → 🔍 Scans tab.
  2. In the "Select Compliance Config" dropdown, pick a config (e.g., "mcpeverything (SOX)").
  3. Verify the config summary bar appears showing framework badge, sample rate, target, and retention.
  4. Click the "▶ Run Scan" button.
  5. Wait for the scan to complete (loading spinner appears).
  6. Verify the Scan Results card appears with results.

📌 Via API

curl -X POST $BASE_URL/compliance/configs/<config-id>/run \
  -H "Content-Type: application/json" \
  -H "$AUTH" \
  -d '{ "samples": null }'

Objective: Create a new custom rule with sample data for the current framework.

  3. Fill in the form as shown below.
  4. Add PASS and FAIL example data in the Sample Data section.
  5. Click "Create Rule".

curl -X PUT $BASE_URL/compliance/rules \
  -H "Content-Type: application/json" \
  -H "$AUTH" \
  -d '{
    "framework": "sox",
    "name": "Key Rotation Policy",
    "description": "Verify API keys are rotated every 90 days",
    "category": "key_management",
    "severity": "critical",
    "isEnabled": true,
    "evaluationConfig": {

        },
        "fail": {

          "output": "WARNING: 3 keys exceed 90-day rotation policy"

    }
  }'
