00 Prerequisites & Setup

🔧 Environment Requirements
  • AgentShield backend running (local npm run dev or deployed at GCP)
  • PostgreSQL with all migrations applied through 012_reports.sql
  • At least one agent registered (e.g., GPT-4 Analyst or MCP Everything Server)
  • Some audit log data present (generate via Playground or Gateway invocations)
  • Logged in as admin role
  • Dashboard accessible at http://localhost:5173 or production URL
⚠️ Before You Begin The Reports module reads from existing data across all AgentShield tables (audit_log, cost_records, compliance_checks, agents, etc.). Ensure you have real or seed data for the reports to return meaningful results. Empty reports are valid but harder to visually verify chart rendering.

API Base URL

# Local development
BASE_URL=http://localhost:3000/api/v1

# Production (GCP)
BASE_URL=https://<your-cloud-run-url>/api/v1

Authentication Header

# Get token via login
curl -X POST $BASE_URL/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@agentshield.local","password":"admin123"}'

# Use token in subsequent requests
AUTH="Authorization: Bearer <your-jwt-token>"
